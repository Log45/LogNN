Apple backend setup (macOS):

1) Build the extension in Apple profile mode:
   bash ./compile_mlx.sh

2) Optional future native MLX integration flags:
   MLX_INCLUDE_FLAGS="-I/opt/homebrew/include" \
   MLX_LINK_FLAGS="-L/opt/homebrew/lib -lmlx" \
   bash ./compile_mlx.sh

3) Runtime usage:
   - device="mlx" uses the Apple backend dispatch path.
   - Current implementation executes Metal compute kernels in float32 on Apple GPU.
   - Use lognn.is_mlx_native_enabled() and lognn.mlx_dispatch_count() for runtime proof.

4) MPS/Metal spike:
   - mps_spike.mm includes Objective-C++ bootstrap for device availability.
   - Not linked by default in compile scripts.
